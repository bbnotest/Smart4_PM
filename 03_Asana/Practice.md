# 🟧 Day 3 — Asana 실습 (Practice)

> **미션**: 공통 시나리오 [`Pixel Dungeon Run`](../00_Overview/03_Game_Project_Scenario.md)를 Asana에서 **섹션·태스크·서브태스크 + 다중 뷰 + 마일스톤**으로 구성한다.
> 산출물(D3b). 예상 60~80분.

---

## 🎯 최종 결과물

- 섹션 5개 + 태스크 9개 + 서브태스크 3개 + 커스텀필드(Priority) + 마일스톤 2개 이상을 갖춘 프로젝트.
- List·Board·Calendar 세 뷰에서 모두 정상 표시.

---

## 📋 과제 단계

### Task 1. 프로젝트 & 섹션
- [ ] 빈 프로젝트 `Pixel Dungeon Run`(List 레이아웃)
- [ ] 섹션 5개: `E2 코어 플레이` `E3 던전·콘텐츠` `E5 UI/UX` `E6 오디오` `E7 QA·출시`

### Task 2. 태스크 9개 (섹션 배치)

| 태스크 | 섹션 | 담당 | 마감 | Priority |
|---|---|:--:|:--:|:--:|
| US-01 자동 전진 | E2 | DEV | 7/08 | High |
| US-02 점프(탭) | E2 | DEV | 7/09 | High |
| US-03 슬라이드 | E2 | DEV | 7/10 | High |
| US-04 충돌/게임오버 | E2 | DEV | 7/11 | High |
| US-05 절차적 생성 | E3 | DEV | 7/24 | High |
| US-06 점수 집계 | E3 | DEV | 7/26 | Med |
| US-07 결과 화면 | E5 | ART | 8/05 | Med |
| US-08 효과음 | E6 | ART | 7/15 | Low |
| US-09 프로토 빌드 | E7 | DEV | 7/17 | High |

- [ ] 태스크 9개 생성 + 섹션 배치
- [ ] **커스텀 필드 `Priority`**(드롭다운) 추가 후 각 태스크에 지정
- [ ] 담당자·마감일 입력

### Task 3. 서브태스크
- [ ] **US-05**에 서브태스크 3개: `바닥 생성` / `플랫폼 배치` / `난이도 점증`

### Task 4. 마일스톤
- [ ] `M1 프로토타입`(7/17), `M2 알파`(7/31)를 **Add milestone**으로 생성

### Task 5. 뷰 전환 확인
- [ ] **Board** 뷰에서 섹션이 칸반 컬럼으로 보이는지 확인 + 태스크 1개를 다른 섹션으로 드래그
- [ ] **Calendar** 뷰에서 마감일들이 달력에 분포하는지 확인

> 완성본을 [List 뷰 목업](../assets/asana_list_mockup.svg)과 비교하세요.

## 🚀 도전 과제 (선택)

- **Rules 자동화**: "Done 섹션으로 이동 → 완료 표시" 규칙 생성
- **Timeline 체험**: 14일 체험판을 켜고 Timeline에서 의존성 1개 연결 후 캡처(체험 종료 전)
- **My Tasks**: 좌측 My Tasks에서 본인 담당 작업만 모아보기

---

## ⚠️ 흔한 실수

- 섹션 대신 새 프로젝트를 만듦 → 같은 게임은 **한 프로젝트 + 섹션**.
- 커스텀 필드를 안 만들고 우선순위를 제목에 적음 → **필드**로 관리해야 정렬·필터 가능.
- 마일스톤을 일반 태스크로 생성 → **Add milestone**을 사용.
