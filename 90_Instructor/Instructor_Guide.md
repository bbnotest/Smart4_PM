# 👨‍🏫 강사용 진행 가이드 (Instructor Guide)

> 이 문서는 **강사가 수업을 운영**하기 위한 일자별 진행안·타이밍·교수법·함정 대응을 담습니다. 학생용 자료는 각 Day 폴더의 `Guide.md`/`Practice.md`입니다.

---

## 1. 과정 운영 원칙

- **오전 = 따라하기(I do → We do)**, **오후 = 직접하기(You do)**. 강사가 시연하면 학생이 따라 하고, 오후엔 스스로 결과물을 만든다.
- **공통 시나리오 고정**: 모든 실습은 [`Pixel Dungeon Run`](../00_Overview/03_Game_Project_Scenario.md). 같은 데이터를 4툴에 넣어 **비교 학습**을 극대화.
- **비교를 계속 상기**: "이거 Trello에선 뭐였죠?" 같은 로제타 질문을 매 Day 반복.
- **정직한 한계 교육**: 무료 플랜의 빈틈(Asana Gantt 유료, Redmine 칸반 없음)을 숨기지 말고 **툴 선택의 근거**로 전환.

---

## 2. 사전 준비 체크리스트 (강사)

- [ ] 4개 툴 **본인 계정**으로 전체 실습을 1회 완주(특히 Jira 팀관리형, Redmine Docker)
- [ ] **Redmine Docker를 강의 PC에서 미리 실행 테스트** — 첫날 가장 큰 리스크
- [ ] 교육장 **네트워크/방화벽**에서 trello/atlassian/asana 접속 및 `docker pull` 가능 여부 확인
- [ ] 프로젝터로 글자 크기 확인(툴 UI는 작음 → 화면 확대/줌)
- [ ] 학생 **계정 생성 안내**(Day 0 체크리스트) 사전 공지
- [ ] 조 편성(3~4인) — 협업자 한도(Trello/Jira/Asana 10명) 안에서 한 명의 워크스페이스에 모이기
- [ ] 각 Day `Guide.md`의 **🖼️ 스크린샷 자리표시**에 실제 공식 화면을 캡처해 삽입(선택, 권장)

## 4. 교수법 팁

- **I do / We do / You do**: 시연 → 함께 → 혼자. 오전엔 절대 혼자 두지 말 것.
- **페어/조별**: 막힘 해소가 빠르고 협업 감각도 학습. 단, 결과물은 개인 제출.
- **비교 질문 루틴화**: 매 툴마다 "이건 어제 그 툴에서 ___였다"를 말하게.
- **함정 선공지**: 위 '흔한 함정'을 **시작 전에** 알려주면 진도 손실이 크게 준다.
- **공식 문서 인용**: 각 Guide 하단 공식 링크를 함께 띄워 "근거 있는 학습" 습관화.
