> 🎯 **개요** — 에픽을 **스토리**로 쪼개 백로그를 채우고 포인트·담당을 넣은 뒤, **담당자에게 전달(코멘트·@멘션)** 합니다.

## A. 스토리 9개 만들기

1. 상단 **`백로그`(Backlog)** 탭 열기
2. **Backlog** 섹션의 **`+ 만들기`(+ Create)** 로 아래 9개 입력 (US-09만 타입 Task):

| 요약 | 타입 | 에픽 | 포인트 | 담당 |
|---|---|---|:--:|:--:|
| US-01 플레이어 이동 | Story | E2 | 3 | DEV |
| US-02 공격 입력 | Story | E2 | 2 | DEV |
| US-03 이동 처리 | Story | E2 | 2 | DEV |
| US-04 턴제 전투(HP) | Story | E2 | 3 | DEV |
| US-05 던전 생성 | Story | E3 | 5 | DEV |
| US-06 적 추적 AI | Story | E3 | 2 | DEV |
| US-07 게임오버 화면 | Story | E5 | 3 | ART |
| US-08 효과음 | Story | E6 | 2 | ART |
| US-09 프로토 빌드 | **Task** | E7 | 3 | DEV |

> 💡 **새 항목이 자동으로 `작업(Task)` 타입으로 만들어질 수 있어요.** 그러면 항목을 열고 **제목 줄(또는 패널 상단)의 타입 아이콘 클릭 → `스토리(Story)`** 로 바꾸세요. (US-09만 `작업/Task` 그대로 둡니다.)

## B. 에픽 연결 + 포인트 + 담당 + 레이블

각 이슈를 클릭하면 오른쪽에 **상세 패널**이 열립니다. 여기서 채웁니다:

- **`에픽`(Epic)** 필드에서 해당 에픽 선택 — *백로그에서 이슈 행에 마우스를 올리면 뜨는 **`+ 에픽`** 버튼으로 이슈를 열지 않고 더 빠르게 지정할 수도 있어요*
- **`Story point estimate`**(포인트)와 **`담당자`(Assignee)** 입력
- **`레이블`(Labels)** 에 분류 꼬리표 1~2개 (바로 아래에서 설명)

백로그에 **스토리 9개**를 만든 모습입니다 👇 (에픽·포인트·담당은 위 **B**처럼 각 이슈를 열어 채워 넣습니다)

![Jira 3단계 — 백로그(실제 화면)](jira_step3.svg)

> 🙋 **에픽 연결을 빼먹으면** Timeline·필터가 비어 보입니다. **모든 스토리에 에픽** 꼭!
> 🙋 **포인트 칸이 없으면** Project settings → Features → **Estimation** 켜기.

> 📷 위 그림은 데모 사이트에서 직접 캡처한 실제 화면입니다

## 🏷️ 레이블로 한 번 더 분류 (+ 우선순위)

에픽이 "큰 묶음"이라면, **레이블(Label)** 은 에픽을 가로지르는 **작은 검색용 꼬리표**입니다. 한 이슈에 여러 개 달 수 있어요. (색이 아니라 **태그**입니다 — 상세 패널 `레이블` 필드에 단어를 입력하고 Enter.)

| 종류 | 예시 | 쓰임 |
|---|---|---|
| 기능 영역 | `combat` · `dungeon` · `ui` · `audio` | "전투 관련만 모아 보기" |
| 직군 | `art` · `sound` · `design` | 직군별로 모아 보기 |
| 급함 표시 | `hotfix` · `blocked` | 급한 것 · 막힌 것 |

- 띄어쓰기 없이 적습니다: `enemy-ai`, `boss-fight` 처럼.
- 예: `US-05 던전 생성` → `dungeon`, `art` 두 개.

### 🚦 우선순위는 어떻게 넣나요?

**뒤의 8·10단계가 우선순위를 전제**하니 지금 채워 둡니다. 무료 **팀 관리(Team-managed)** 프로젝트는 처음엔 `우선순위`(Priority) 필드가 숨어 있을 수 있어요.

- **정식(권장)**: `프로젝트 설정 → 기능(Features)` 에서 **우선순위(Priority)** 켜기 → 값은 **Highest·High·Medium·Low**. 그러면 8단계에서 `priority = Highest` 검색이 됩니다.
- **간단**: 설정을 못 켜면 `p1`·`p2`·`p3` **레이블**로 대신합니다 (검색은 `labels = p1`).

> 🙋 어느 쪽이든 "무엇부터"가 **데이터로** 남아야 **[8단계 JQL](Step8.md)** 에서 급한 것만 추려지고, **[10단계 대시보드](Step10.md)** 에서 분포 차트가 그려집니다.

## 🔍 이슈 하나 열어보기 — 상세 패널 한눈에

이슈를 클릭하면 오른쪽 **상세 패널**에 그 이슈의 정보가 모두 모여 있습니다. 한 바퀴 둘러볼게요:

| 영역 | 필드 | 무엇 |
|---|---|---|
| 본문 | **설명(Description)** | "무엇을·왜"를 자세히. 표·체크리스트·이미지도 가능 |
| 사람 | **담당자 · 보고자** | 맡은 사람 · 만든 사람 |
| 분류 | **에픽(부모) · 레이블** | 큰 묶음 · 작은 꼬리표 |
| 추정·일정 | **스토리 포인트 · 시작 날짜 · 기한** | 크기 · 언제부터·언제까지 |
| 진행 | **상태(To Do·진행 중·완료)** | 지금 어디까지 |

> 🙋 **설명(Description)을 비워 두지 마세요.** 제목만으론 "무엇을"까지만 보입니다. 본문에 **수용 기준(이게 되면 끝)** 한두 줄을 적으면 개발이 헤매지 않습니다.

## C. 작업을 담당자에게 전달 — 배정·코멘트·@멘션

만든 **작업(Task/Story)** 은 사람에게 전달돼야 굴러갑니다. Jira의 기본 전달 방식:

1. **담당자(Assignee) 지정** → 그 사람에게 **알림**이 갑니다("당신 일입니다").
2. 작업 상세 하단 **코멘트**에 요청·맥락을 남기고 **`@이름`(멘션)** → 콕 집어 전달.
   - 예: `US-05 던전 생성`에 코멘트 `@아트 던전 타일셋 먼저 필요해요` → 아트팀에 알림.
3. 배정·@멘션 알림은 상대의 **알림함**에 쌓여 "내가 볼 일"이 분명해집니다.

> 🙋 **전달 = 배정 + 코멘트·@멘션**. 말로 "해주세요" 하는 대신 작업에 남기면 **기록**으로 남습니다.
> 🔸 여기서 "작업"은 **계획된 개발 항목**입니다. **QA가 발견해 등록하는 버그 '이슈'** 는 결이 다른 별도 프로세스예요 → **[7단계 · QA·이슈 관리](Step7.md)**

## 🎮 현장 감각 — 게임 PM은 이렇게

> **Pixel Dungeon 맥락**<br>
> 스토리 포인트는 '며칠 걸린다'는 시간이 아니라, '얼마나 까다로운가'를 서로 비교한 점수입니다.<br>
> '던전 생성(5점)'이 '효과음(2점)'보다 점수가 높은 건, 던전을 매번 다르게 자동으로 만드는 일이 변수가 훨씬 많기 때문입니다.<br>
> 개발(DEV)·아트(ART)로 담당을 나눠 두면 둘이 동시에 일할 수 있습니다.

**⚠️ 흔한 실수**
- 포인트를 '며칠'처럼 **시간**으로 착각 → 팀이 합의한 **상대 크기**로.
- 스토리 제목을 단순 기능명으로 → **유저 가치(US-xx)** 형태로 적으면 우선순위가 또렷해짐.

**🎤 면접 한 줄**
> *"백로그를 **'사용자가 무엇을 한다'(유저 스토리)** 형식으로 작성하고, **서로 비교한 점수(스토리 포인트)** 로 크기를 추정해 한 스프린트에 담을 양을 가늠했습니다."*
