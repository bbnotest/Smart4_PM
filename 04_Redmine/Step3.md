# 🟥 Redmine · 3단계 — 이슈와 WBS (상위/하위)

> 🎯 이번 단계 목표: **상위 이슈 아래 하위 이슈를 만들어 WBS를 구성한다.**
> 📍 [← 2단계](Step2.md) · 다음 [4단계 →](Step4.md)

---

Redmine엔 "에픽"이란 단어는 없지만, **상위 이슈 아래 하위 이슈**를 두면 똑같이 WBS가 됩니다.

```mermaid
graph TD
    P100["#100 E2 코어 플레이 (상위)"] --> C1["#1 US-01 플레이어 이동"]
    P100 --> C2["#2 US-02 공격 입력"]
    P100 --> C4["#4 US-04 턴제 전투(HP)"]
    P200["#200 E3 던전·콘텐츠 (상위)"] --> C5["#5 US-05 던전 생성"]
```

## A. 상위 이슈(에픽 역할) 먼저

1. **`New issue`** → Tracker = `Feature`, Subject = `E2 코어 플레이` → Create
2. 같은 방식으로 `E3 던전·콘텐츠` 도

## B. 하위 이슈

1. **`New issue`** → Subject `US-01 플레이어 이동`
2. **`Parent task`**(상위 작업) 칸에 `E2 코어 플레이` 지정 ← 핵심!
3. US-01~04를 E2 아래, US-05~06을 E3 아래에 둡니다

> 🖼️ 공식 스크린샷 자리 — 이슈 + Parent task

> ✅ 상위 이슈를 열면 아래에 **Subtasks 목록**이 보입니다 = WBS 완성.

---

## 🎮 현장 감각 — 게임 PM은 이렇게

> **Pixel Dungeon 맥락**<br>
> Redmine엔 '에픽'이라는 말이 없지만, 상위 이슈 아래 하위 이슈를 두면 똑같이 WBS가 됩니다.<br>
> 핵심은 하위 이슈의 'Parent task' 한 칸을 지정하는 것입니다.<br>
> 도구 이름이 달라도 "큰 것을 작은 것으로 쪼갠다"는 원리는 어디서나 같습니다.

**⚠️ 흔한 실수**
- Parent task를 안 지정함 → 평면 목록이 되어 WBS가 안 됩니다.
- 상위 이슈에 직접 작업을 함 → 상위는 묶음, 실제 작업은 하위 이슈에 둡니다.
