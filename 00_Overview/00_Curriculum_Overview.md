# 📘 과정 개요 (Curriculum Overview)

> 게임 개발 PM 지망생을 위한 **협업툴 4종(Jira·Asana·Redmine·Trello) 1주 집중 실습** 과정의 마스터 플랜입니다.

---

## 1. 과정 정보

| 항목 | 내용 |
|---|---|
| **대상** | 게임 개발 PM 지망 학생 (WBS·Gantt·Kanban **개념은 이해**, 현업 **툴 경험은 없음**) |
| **기간** | 5일 (월~금), 1일 약 6시간 기준 — 학원 일정에 맞춰 조정 가능 |
| **형식** | 오전 *가이드 따라하기* + 오후 *결과물 실습* (Hands-on) |
| **결과물** | 툴별 산출물 4종 + 캡스톤 1종 → **포트폴리오** |
| **언어/환경** | 한국어 교재 · 웹 브라우저(Chrome 권장) · Redmine은 Docker |

---

## 2. 학습 목표 & 성취기준

학생은 과정을 마치면 **스스로** 다음을 할 수 있어야 합니다(성취기준 = 평가 가능한 행동).

1. **계정·환경 구축** — 4개 툴의 계정을 만들고(또는 Redmine을 설치하고) 프로젝트를 생성한다.
2. **WBS 구현** — 게임 프로젝트의 작업분해를 각 툴의 계층(에픽/스토리/서브태스크 또는 상위/하위 이슈)으로 표현한다.
3. **Kanban 운영** — 워크플로 컬럼이 있는 보드를 만들고 작업 카드를 상태에 따라 이동한다.
4. **Gantt/일정 관리** — 시작·마감·의존성을 가진 일정(Timeline/Gantt)을 작성한다.
5. **진행 추적** — 스프린트/버전/마일스톤과 리포트로 진행 상황을 추적한다.
6. **툴 선택** — 상황(팀 규모·예산·방식)에 맞는 툴을 **근거와 함께** 선택한다.

---

## 3. 선수 지식 (이미 갖춘 것으로 가정)

- WBS(작업분해구조), Gantt 차트, Kanban 보드의 **개념**
- 게임 개발 파이프라인(기획→아트→개발→QA→출시)에 대한 기본 이해
- 스프린트/백로그/마일스톤 같은 용어에 대한 막연한 친숙함

> 개념 복습이 필요하면 [`01_PM_Concepts_to_Tools.md`](01_PM_Concepts_to_Tools.md)의 1절을 먼저 읽으세요.

## 4. 🧰 Day 0 — 시작 전 준비 체크리스트

수업 첫날 전에(또는 1교시에) 아래를 준비합니다. **계정 생성에 쓸 이메일 1개**를 정해 4개 툴에 동일하게 사용하면 관리가 쉽습니다.

- [ ] **이메일 계정** (가급적 Google 계정 — 소셜 로그인에 편리)
- [ ] **Chrome**(또는 Edge) 최신 브라우저
- [ ] **Trello** 계정 — https://trello.com/signup (무료)
- [ ] **Atlassian/Jira** 계정 — https://www.atlassian.com/software/jira/free (무료, 사용자 10명)
- [ ] **Asana** 계정 — https://asana.com (무료 Personal)
- [ ] **Redmine 실습 환경** — 아래 셋 중 하나 ([상세](../04_Redmine/Guide.md))
  - [ ] (권장) **Docker Desktop** 설치 후 공식 이미지로 로컬 실행
  - [ ] 학원에서 제공하는 **공용 Redmine 서버** 접속 정보
  - [ ] 무료 **호스팅형 Redmine** 체험 계정
- [ ] (선택) 각 툴 **모바일 앱** 설치 — 알림/현업 감각 체험용

> ⚠️ **개인정보 주의**: 실습은 가짜/샘플 데이터로 진행합니다. 실제 회사 기밀이나 타인의 개인정보를 입력하지 마세요.

## 5. 🗓️ 5일 상세 일정

> 매일 **오전 = 가이드(따라하기)**, **오후 = 실습(결과물 제작)**. 모든 실습은 공통 시나리오 [`03_Game_Project_Scenario.md`](03_Game_Project_Scenario.md)를 사용합니다.

| Day | 주제 | 오전 (Guide) | 오후 (Practice) | 산출물 |
|:--:|---|---|---|---|
| **1** | 오리엔테이션 + **Trello** | 과정 OT · 개념→툴 매핑 · 비교표 · 공통 시나리오 숙지 · Trello 기본(보드/리스트/카드/라벨/체크리스트/마감/Butler) | 게임 프로젝트 **Kanban 보드** 구축 + 자동화 1개 | Trello 스프린트 보드 |
| **2** | **Jira ①** | 계정·사이트·스크럼 프로젝트 생성 · 이슈 계층(Epic→Story/Task→Sub-task) · 백로그 | **백로그(=WBS)** 작성 + 스토리 포인트 + **스프린트** 생성·시작 | Jira 제품 백로그 + Sprint 1 |
| **3** | **Jira ② + Asana** | Jira **Timeline**·보드 운영·번다운/리포트·JQL 기초 / Asana 프로젝트·태스크·서브태스크·뷰(List/Board/Calendar)·마일스톤 | Jira Timeline에 에픽 일정·의존성 / Asana 프로젝트 + 마일스톤 | Jira Timeline · Asana 프로젝트 |
| **4** | **Redmine** | Docker 설치(or 데모) · 관리자 기본 · 프로젝트/트래커/이슈 · **상위·하위 이슈(WBS)** · 멤버/역할 | **버전(마일스톤)+로드맵** · 시작/마감/진행률 → **내장 Gantt** · 시간 기록 | Redmine 프로젝트 + Gantt |
| **5** | **비교 + Capstone** | 4툴 비교 토론 · **툴 선택 의사결정** 연습 · 캡스톤 착수 | 상황에 맞는 **툴 선택 + 근거** + 미니 계획 수립 · **발표 & 평가** | 툴 선택 보고서 + 캡스톤 계획 |

### 하루 시간 배분 예시 (6시간 기준)

```
09:00–09:15  복습 & 오늘의 목표
09:15–11:00  오전 가이드(강사 시연 + 학생 따라하기)
11:00–12:00  오전 실습 착수
13:00–15:30  오후 실습(결과물 제작)
15:30–16:30  결과물 점검 · 체크리스트 자기평가
16:30–17:00  Q&A · 회고 · 내일 예고
```

> 일정이 4일이면 Day3의 Asana를 Day4 오전으로 이동, 6~7일이면 각 툴에 0.5~1일씩 추가해 심화 실습을 넣습니다.

## 7. 📝 평가 개요

- 평가 기준 전체는 [`90_Instructor/Assessment_Rubric.md`](../90_Instructor/Assessment_Rubric.md) 참고.
- 권장 배점: 툴별 산출물 4종 각 **15%**(합 60%) + 캡스톤 **30%** + 참여/회고 **10%**.
- 각 산출물은 **자기평가 체크리스트 → 동료 리뷰 → 강사 루브릭 평가** 3단계.

## 8. 무료 요금제 정책 & 보완책 (정직한 안내)

| 부족한 점 | 어떤 툴에서 | 보완책 |
|---|---|---|
| **Gantt가 유료** | Asana(Timeline은 Starter), Trello(Timeline은 Premium) | Jira Timeline·Redmine 내장 Gantt로 Gantt 역량을 채우고, Asana/Trello는 무료 뷰(List/Calendar/Board)로 학습 + 필요 시 **14일 체험판** |
| **Kanban이 코어에 없음** | Redmine | 상태 워크플로 + 필터로 의사 Kanban 실습, 심화 시 **Redmine Agile 플러그인** 소개(설치는 선택) |
| **사용자/보드 제한** | Trello(보드10·협업자10), Jira(10명), Asana(10명) | 학습용으로 충분. 팀 실습은 **3~4인 1조**로 구성 |

> 핵심 메시지: **"하나의 무료 툴이 모든 걸 다 해주지 않는다."** 이 빈틈을 이해하는 것이 PM의 **툴 선택 역량**입니다. → [`02_Tool_Comparison_Matrix.md`](02_Tool_Comparison_Matrix.md)
