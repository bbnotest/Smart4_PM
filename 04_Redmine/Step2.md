# 🟥 Redmine · 2단계 — 관리자 설정 & 프로젝트

> 🎯 **개요** — 자체 호스팅 도구는 PM이 **관리자**도 겸합니다. 기본 구성을 불러오고, 우리 프로젝트 **`Pixel Dungeon`** 을 만듭니다.

## A. 관리자 초기 설정 (기본 구성 불러오기)

1. **admin**으로 로그인한 상태에서 상단 **`관리자`(Administration)** 진입
2. 트래커·상태·역할이 비어 있으면 **`기본 구성 불러오기`(Load the default configuration)** 클릭
   → 기본 **트래커(Bug·Feature·Support)** · **상태(New·In Progress·Resolved·Feedback·Closed·Rejected)** · 역할·우선순위가 한 번에 생깁니다
3. (선택) `관리자 → 설정(Settings) → 표시(Display)` 에서 **언어를 한국어**로

> 🙋 **이걸 안 하면** 트래커가 없어서 이슈를 아예 못 만듭니다. 서버를 새로 띄웠다면 **꼭 한 번** 눌러주세요.

## B. 프로젝트 만들기 + 모듈 켜기

1. 상단 **`프로젝트`(Projects) → `새 프로젝트`(New project)**
2. **이름(Name)** `Pixel Dungeon` 입력 (식별자 identifier는 자동 생성)
3. **모듈(Modules)** 체크 — 뒤 단계에서 쓸 기능들입니다(중요!):

| 모듈 | 쓰는 단계 |
|---|---|
| **이슈 추적**(Issue tracking) | 3~9단계 전부 |
| **간트**(Gantt) | 6단계 |
| **로드맵**(Roadmap) | 5단계 |
| **시간 추적**(Time tracking) | 8단계 |
| **위키**(Wiki) · **소식**(News) | 9단계 |
| 달력(Calendar) | 보조 |

4. **트래커(Trackers)** 에서 `Bug`·`Feature`·`Support` 체크 → **`만들기`(Create)**

> 🙋 나중에 **간트 탭이 안 보이면** 이 모듈 체크를 빠뜨린 거예요 → `프로젝트 설정(Settings) → 모듈(Modules)` 에서 다시 켜면 됩니다.

## C. 팀원 초대 (멤버 & 역할)

- `프로젝트 설정 → 멤버(Members) → 새 멤버`(New member) → 사용자 선택 → **역할**(관리자/개발자/보고자) 지정
- 역할이 **권한**을 정합니다: 예) `보고자(Reporter)`는 이슈 등록만, `개발자(Developer)`는 상태 변경까지.

> 💡 혼자 연습 중이면 지금은 건너뛰어도 됩니다. 다만 "역할로 권한을 나눈다"는 개념은 알아두세요.

## 🎮 현장 감각 — 게임 PM은 이렇게

> **Pixel Dungeon 맥락**<br>
> 자체 호스팅 도구는 PM이 관리자 역할까지 겸하는 경우가 많습니다.<br>
> 그래서 트래커·상태·역할 같은 기본 구성을 직접 불러오는 첫 세팅이 중요합니다.<br>
> 프로젝트를 만들 때 켜는 모듈(간트·로드맵)이 뒤 단계에서 쓸 기능을 좌우합니다.

**⚠️ 흔한 실수**
- 기본 구성을 안 불러옴 → 트래커가 없어 이슈를 못 만듭니다. 꼭 한 번 누릅니다.
- 간트·로드맵 모듈 체크를 빠뜨림 → 나중에 그 탭이 안 보입니다.

**🎤 면접 한 줄**
> *"관리자 설정부터 **모듈 구성**까지 직접 잡아, 프로젝트 환경을 처음부터 세팅했습니다."*
