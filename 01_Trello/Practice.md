# 🟦 Day 1 — Trello 실습 (Practice)

> **미션**: 공통 시나리오 [`Pixel Dungeon Run`](../00_Overview/03_Game_Project_Scenario.md)의 **Sprint 1 Kanban 보드**를 직접 완성한다.
> 가이드를 보며 따라 하는 단계는 끝났습니다. 이제 **스스로** 만듭니다. (예상 60~90분)

---

## 🎯 최종 결과물

- 5개 리스트(워크플로) + 9개 카드(US-01~09) + 라벨(에픽) + 체크리스트 1개 이상 + 마감일 + Butler 규칙 1개를 갖춘 Trello 보드 1개.

---

## 📋 과제 단계

### Task 1. 보드 & 워크플로
- [ ] 보드 생성: `Pixel Dungeon Run - Sprint 1`
- [ ] 리스트 5개: `Backlog` · `To Do` · `In Progress` · `Review` · `Done`

### Task 2. 카드 9개 입력 (Backlog에 모두 생성)
시나리오 Sprint 1 백로그를 그대로 사용합니다.

| 카드 제목 | 라벨(에픽) | 포인트 |
|---|---|:--:|
| US-01 플레이어 자동 전진 | E2 코어(파랑) | 3 |
| US-02 점프(탭) | E2 코어(파랑) | 2 |
| US-03 슬라이드(길게) | E2 코어(파랑) | 2 |
| US-04 충돌/게임오버 | E2 코어(파랑) | 3 |
| US-05 절차적 바닥/플랫폼 생성 | E3 던전(초록) | 5 |
| US-06 점수 집계(거리·코인) | E3 던전(초록) | 2 |
| US-07 게임오버 결과 화면 | E5 UI(보라) | 3 |
| US-08 핵심 효과음 | E6 오디오(주황) | 2 |
| US-09 1분 플레이 프로토타입 빌드 | E7 출시(빨강) | 3 |

- [ ] 카드 9개 생성 완료
- [ ] 각 카드에 **라벨**(에픽 색) 지정
- [ ] 포인트는 카드 제목 끝에 `(3pt)`처럼 표기하거나, 무료 Power-Up(Story Points) 사용

### Task 3. 카드 디테일
- [ ] **US-05**에 체크리스트 추가: `바닥 생성` / `플랫폼 배치` / `난이도 점증` (1개 완료 체크 → ☑1/3)
- [ ] **US-09**에 마감일 `2026-07-17` 지정
- [ ] 카드 3개 이상에 담당자(DEV/ART/QA) 배정

### Task 4. 칸반 운영(상태 이동)
- [ ] US-09 → `Done`, US-07 → `Review`, US-04 → `In Progress`로 드래그
- [ ] US-01·02·03 → `To Do`로 이동 (나머지는 Backlog 유지)

> 이 배치는 목업 그림과 동일합니다. 완성 후 [목업](../assets/trello_board_mockup.svg)과 비교해 보세요.

### Task 5. Butler 자동화
- [ ] 규칙 생성: **"카드가 `Done`으로 이동하면 → 마감일 완료 표시"**
- [ ] (도전) `In Progress`로 이동 시 → 카드에 `🔨` 라벨 또는 현재 날짜 코멘트 추가

## ✅ 완성 기준 체크리스트 (자기평가)

| # | 기준 | 충족 |
|:--:|---|:--:|
| 1 | 리스트 5개가 워크플로 순서대로 있다 | ☐ |
| 2 | 카드 9개가 모두 존재한다 | ☐ |
| 3 | 모든 카드에 에픽 라벨이 있다 | ☐ |
| 4 | 체크리스트가 1개 이상, 마감일이 1개 이상 있다 | ☐ |
| 5 | 카드가 상태에 맞게 분산 배치되어 있다 | ☐ |
| 6 | Butler 규칙이 1개 동작한다 | ☐ |
| 7 | 보드만 봐도 "지금 무엇이 진행 중"인지 읽힌다 | ☐ |

7개 중 **6개 이상**이면 통과.

## 📤 제출물

1. 보드 전체가 보이는 **스크린샷 1장** (또는 보드 공유 링크: 보드 메뉴 → Share → 링크)
2. Butler 규칙 화면 **스크린샷 1장**
3. 한 줄 회고: "Trello가 쉬웠던 점 / 아쉬웠던 점"

> 파일명 예: `D1_Trello_홍길동.png`

## 🚀 도전 과제 (선택)

- **WIP 제한**: List Limits Power-Up으로 `In Progress`를 3개로 제한
- **Calendar**: Calendar Power-Up을 켜고 마감일을 달력에서 확인
- **스프린트 2 준비**: 미완료 카드를 새 리스트 `Sprint 2`로 옮기는 규칙 만들기

---

## ⚠️ 흔한 실수

- 카드를 리스트로 만들어 버림 → 작업은 **카드**, 단계는 **리스트**.
- 라벨을 색만 쓰고 이름을 안 지정 → 나중에 무슨 에픽인지 모름. **이름 필수**.
- Butler 규칙의 Trigger/Action을 혼동 → "언제(Trigger) → 무엇을(Action)" 순서 기억.
