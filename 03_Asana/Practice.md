# 🟧 Asana 직접 해보기 (연습)

> [가이드](Guide.md)를 봤으니, 같은 예제 [Pixel Dungeon](../00_Overview/03_Game_Project_Scenario.md)로 **섹션·태스크·서브태스크 + 다중 뷰 + 마일스톤**을 직접 만들어 보세요.

---

## 🎯 만들 것

섹션 5개 + 태스크 9개 + 서브태스크 3개 + 커스텀필드(Priority) + 마일스톤 2개를 갖춘 프로젝트. List·Board·Calendar 세 뷰에서 모두 정상 표시.

---

## ✋ 따라 만들기

### 1. 프로젝트 & 섹션
- 빈 프로젝트 `Pixel Dungeon`(List 레이아웃)
- 섹션 5개: `E2 코어 플레이` · `E3 던전·콘텐츠` · `E5 UI/UX` · `E6 오디오` · `E7 QA·출시`

### 2. 태스크 9개 (섹션 배치)

| 태스크 | 섹션 | 담당 | 마감 | Priority |
|---|---|:--:|:--:|:--:|
| US-01 플레이어 이동 | E2 | DEV | 7/08 | High |
| US-02 공격 입력 | E2 | DEV | 7/09 | High |
| US-03 이동 처리 | E2 | DEV | 7/10 | High |
| US-04 턴제 전투(HP) | E2 | DEV | 7/11 | High |
| US-05 던전 생성 | E3 | DEV | 7/24 | High |
| US-06 적 추적 AI | E3 | DEV | 7/26 | Med |
| US-07 게임오버 화면 | E5 | ART | 8/05 | Med |
| US-08 효과음 | E6 | ART | 7/15 | Low |
| US-09 프로토 빌드 | E7 | DEV | 7/17 | High |

- 태스크 9개 생성 + 섹션 배치 + 담당·마감 입력
- **커스텀 필드 `Priority`**(드롭다운) 추가 후 각 태스크에 지정

### 3. 서브태스크
- **US-05**에 서브태스크 3개: `방 생성` / `복도 연결` / `계단 배치`

### 4. 마일스톤
- `M1 프로토타입`(7/17), `M2 알파`(7/31)를 **Add milestone**으로 생성

### 5. 뷰 전환 확인
- **Board** 뷰에서 섹션이 칸반 컬럼으로 보이는지 확인 + 태스크 1개를 다른 섹션으로 드래그
- **Calendar** 뷰에서 마감일이 달력에 분포하는지 확인

> 다 만들었으면 [List 뷰 목업](../assets/asana_list_mockup.svg)과 비교해 보세요.

---

## ✅ 스스로 점검

- [ ] 섹션 5개가 있다
- [ ] 태스크 9개가 섹션에 배치돼 있다
- [ ] Priority 커스텀 필드가 모든 태스크에 있다
- [ ] 서브태스크 3개가 US-05에 있다
- [ ] 마일스톤이 2개 이상 ◆로 표시된다
- [ ] List·Board·Calendar 세 뷰가 모두 정상

---

## 🚀 더 해보기 (선택)

- **Rules 자동화**: "Done 섹션으로 이동 → 완료 표시" 규칙 생성
- **Timeline 체험**: 14일 체험판을 켜고 Timeline에서 의존성 1개 연결 후 캡처
- **My Tasks**: 본인 담당 작업만 모아보기

> 💡 List·Board·Calendar 화면을 **캡처해두면 면접 자료**로 좋습니다.

---

## ⚠️ 흔한 실수

- 섹션 대신 새 프로젝트를 만듦 → 같은 게임은 **한 프로젝트 + 섹션**.
- 우선순위를 제목에 적음 → **커스텀 필드**로 관리해야 정렬·필터 가능.
- 마일스톤을 일반 태스크로 생성 → **Add milestone** 사용.
