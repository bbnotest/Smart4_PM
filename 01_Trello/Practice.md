# 🟦 Trello 직접 해보기 (연습)

> [가이드](Guide.md)를 봤으니, 이제 **직접** 만들어볼 차례입니다. 같은 게임 예제 [Pixel Dungeon](../00_Overview/03_Game_Project_Scenario.md)로, 보지 않고 스스로 재현해 보세요.

---

## 🎯 만들 것

5개 리스트(워크플로) + 9개 카드 + 라벨(에픽) + 체크리스트 1개 + 마감일 + Butler 규칙 1개를 갖춘 Trello 보드.

---

## ✋ 따라 만들기

### 1. 보드 & 워크플로
- 보드 생성: `Pixel Dungeon - Sprint 1`
- 리스트 5개: `Backlog` · `To Do` · `In Progress` · `Review` · `Done`

### 2. 카드 9개 (Backlog에 생성)

| 카드 제목 | 라벨(에픽) | 포인트 |
|---|---|:--:|
| US-01 플레이어 이동(턴제) | E2 코어(파랑) | 3 |
| US-02 공격 입력 | E2 코어(파랑) | 2 |
| US-03 이동 충돌 처리 | E2 코어(파랑) | 2 |
| US-04 턴제 전투(HP) | E2 코어(파랑) | 3 |
| US-05 절차적 던전 생성 | E3 던전(초록) | 5 |
| US-06 적 1종 + 추적 AI | E3 던전(초록) | 2 |
| US-07 사망/게임오버 화면 | E5 UI(보라) | 3 |
| US-08 핵심 효과음 | E6 오디오(주황) | 2 |
| US-09 1층 플레이 프로토타입 빌드 | E7 출시(빨강) | 3 |

- 카드마다 **라벨**(에픽 색) 지정
- 포인트는 제목 끝에 `(3pt)` 처럼 표기

### 3. 카드 디테일
- **US-05**에 체크리스트: `방 생성` / `복도 연결` / `계단 배치` (1개 체크 → ☑1/3)
- **US-09**에 마감일 `2026-07-17`
- 카드 3개 이상에 담당자(DEV/ART/QA) 지정

### 4. 칸반 운영(상태 이동)
- US-09 → `Done`, US-07 → `Review`, US-04 → `In Progress`, US-01·02·03 → `To Do` 로 드래그 (나머지는 Backlog)

### 5. Butler 자동화
- 규칙: **"카드가 `Done`으로 이동하면 → 마감일 완료 표시"**

> 다 만들었으면 [목업 그림](../assets/trello_board_mockup.svg)과 비교해 보세요.

---

## ✅ 스스로 점검

아래에 모두 "네"면 충분합니다.
- [ ] 리스트 5개가 워크플로 순서대로 있다
- [ ] 카드 9개가 모두 있고, 에픽 라벨이 달려 있다
- [ ] 체크리스트·마감일이 각각 1개 이상 있다
- [ ] 카드가 상태에 맞게 흩어져 있다
- [ ] Butler 규칙이 1개 동작한다
- [ ] 보드만 봐도 "지금 무엇이 진행 중"인지 읽힌다

---

## 🚀 더 해보기 (선택)

- **WIP 제한**: List Limits Power-Up으로 `In Progress`를 3개로 제한
- **Calendar**: Calendar Power-Up으로 마감일을 달력에서 보기
- **다음 스프린트 준비**: 미완료 카드를 새 리스트로 옮기는 규칙 만들기

> 💡 완성한 보드를 **캡처해두면 포트폴리오/면접 자료**로 좋습니다.

---

## ⚠️ 흔한 실수

- 카드를 리스트로 만들어 버림 → 작업은 **카드**, 단계는 **리스트**.
- 라벨을 색만 쓰고 이름을 안 지정 → 무슨 에픽인지 모름. **이름 필수**.
- Butler의 Trigger/Action 혼동 → "언제(Trigger) → 무엇을(Action)" 순서.
